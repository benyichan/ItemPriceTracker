# 项目颜色系统文档

## 概述

本项目使用了一个包含12种主题颜色的协调调色板，旨在提供视觉和谐、专业美观的用户界面，同时确保符合可访问性标准。

### 颜色系统原理

1. **色彩和谐**：选择了基于色轮的协调颜色，确保不同颜色之间的视觉和谐
2. **可访问性**：所有颜色都经过调整，确保在浅色和深色模式下都有足够的对比度
3. **一致性**：通过CSS变量和主题系统，确保颜色在整个应用中的一致应用
4. **灵活性**：用户可以根据个人喜好选择不同的主题颜色

## 使用指南

### 主题颜色的应用

1. **主要颜色**：用于按钮、链接、高亮元素和品牌标识
2. **次要颜色**：用于辅助元素、背景和非主要交互元素
3. **状态颜色**：用于成功、警告、错误和信息状态
4. **中性颜色**：用于文本、背景和边框

### 代码中的使用方法

1. **在组件中使用**：
   ```tsx
   // 在 SettingsView.tsx 中选择主题颜色
   <motion.button
     onClick={() => onColorChange('#2A5CAA')}
     style={{
       background: `linear-gradient(135deg, #2A5CAA, #2A5CAA80)`
     }}
   />
   ```

2. **在主题系统中使用**：
   ```typescript
   // 在 useTheme.ts 中定义颜色
   const THEME_COLORS: Record<ThemeColor, { light: string; dark: string }> = {
     '#2A5CAA': { 
       light: '214 62% 41%', 
       dark: '214 70% 55%' 
     },
     // 其他颜色...
   };
   ```

3. **在默认设置中使用**：
   ```typescript
   // 在 db.ts 中设置默认颜色
   export function getDefaultSettings(): Settings {
     return {
       theme: 'system',
       primaryColor: '#2A5CAA',
       // 其他设置...
     };
   }
   ```

## 可访问性合规性

### 对比度标准

- **文本对比度**：所有文本与背景的对比度至少达到 WCAG AA 标准（正常文本 4.5:1，大文本 3:1）
- **UI元素对比度**：所有UI元素与背景的对比度至少达到 WCAG AA 标准（3:1）

### 深色模式支持

- 所有颜色都有对应的深色模式版本，确保在不同主题下都有良好的显示效果
- 深色模式下的颜色经过调整，确保足够的对比度和可读性

### 高对比度模式

- 系统会根据用户的系统设置自动调整颜色对比度
- 所有颜色都支持高对比度模式的自动调整

## 维护指南

### 添加新颜色

1. 在 `useTheme.ts` 中：
   - 更新 `ThemeColor` 类型定义
   - 在 `THEME_COLORS` 对象中添加新颜色的 HSL 值映射

2. 在 `SettingsView.tsx` 中：
   - 在 `THEME_COLORS` 数组中添加新颜色

3. 在 `COLOR_SYSTEM.md` 中：
   - 更新颜色列表和相关文档

### 修改现有颜色

1. 在 `useTheme.ts` 中：
   - 更新对应颜色的 HSL 值映射

2. 在 `COLOR_SYSTEM.md` 中：
   - 更新颜色的相关信息

## 结论

本颜色系统旨在提供一个既美观又实用的调色板，满足不同用户的需求和偏好。通过遵循本指南，开发者可以确保颜色在整个应用中的一致应用，同时保持良好的可访问性和用户体验。
